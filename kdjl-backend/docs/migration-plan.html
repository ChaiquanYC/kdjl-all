四、技术风险 & 缓解措施

高风险
战斗引擎逻辑迁移错误 — 原 PHP 代码无文档，1700+ 行战斗计算有大量隐式规则（五行克制、暴击公式、属性加成）。
缓解：先写战斗引擎的集成对比测试（同样输入 → PHP 和新 Java 输出对比），逐用例验证。
高风险
字符集迁移导致中文乱码 — 混合 latin1/utf8/gbk/utf8mb4 的历史数据。
缓解：逐表分析原始字节编码，使用 HEX 校验 + 抽样人工验证。
高风险
实时聊天系统行为差异 — Comet 长轮询 → WebSocket 语义不同，可能有消息丢失/乱序。
缓解：保留消息序列号，客户端去重 + 重连补拉机制。
中风险
支付/充值系统兼容 — 涉及外部平台 API（4399、webgame 等），接口可能已变更。
缓解：提前梳理所有外部依赖，与平台确认 API 文档。
中风险
前端动画性能 — 原 Flash/简单 JS 动画 → React 动画，需确保 60fps。
缓解：使用 CSS animation + requestAnimationFrame，Canvas 渲染战斗场景（备选方案）。
低风险
MySQL 8.0 语法兼容 — 已识别的问题均可自动/半自动修复。
缓解：编写 SQL 转换脚本，CI 中跑全量 schema 导入验证。
