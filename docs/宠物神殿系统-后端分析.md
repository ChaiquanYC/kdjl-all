## 二、宠物类型体系 (wx 五行)

| wx | 类型 | 可参与的系统 |
|----|------|------------|
| 1-5 | 五行（金木水火土） | 进化、合成、涅磐、成长抽取（需道具 3383） |
| 6 | 神宠 | 进化、合成、涅磐、成长抽取（不需道具 3383） |
| 7 | 神圣 | 神圣进化、神圣转生、成长转化 |

关键判断函数（前端 React 对应）：
```ts
const isDivine = (pet) => pet.wx === 7;   // 神圣
const isSpirit = (pet) => pet.wx === 6;   // 神宠
```

## 三、各功能详细规则

### 3.1 进化 (Tab 1)

**API：** `jhGate.php?n={style}&id={petId}&pids={pids}&bhid={keepCzlItemId}`

**约束：**
- 宠物 wx < 7（非神圣）
- 进化次数 `remaketimes` < 10
- 未被抽取过成长（`player_ext.chouqu_chongwu` 检查）
- 操作间隔 ≥ 2 秒
- 金币 ≥ 1000
- 等级满足 `remakelevel` 要求
- 拥有 `remakepid` 指定的材料道具

**两种进化风格：**

| 风格 | czl 区间 | 成长增量 |
|------|---------|---------|
| n=1 | < 50 | `rand(1,5)/10 + (level-levels)/200` |
| n=1 | 50-80 | `rand(1,3)/10` |
| n=1 | ≥ 80 | `0.1` |
| n=2 | < 50 | `rand(5,10)/10 + (level-levels)/200` |
| n=2 | 50-70 | `rand(4,7)/10` |
| n=2 | 70-80 | `rand(3,5)/10` |
| n=2 | 80-90 | `rand(2,3)/10` |
| n=2 | ≥ 90 | `rand(1,3)/10` |

特殊道具：
- `pids=1221`：追加 `rand(1,3)/10`
- `pids=1222`：追加 `rand(3,6)/10`
- `bhid`（守护道具）：`keepczl:N` 效果，czl 上限受 N 约束（N ≥ 150）

**数据库操作：** 更新 userbb（外观、名称、czl、remakelevel/remakepid/remaketimes），消耗道具，扣除金币。

### 3.2 合成 (Tab 2)

**API：** `cmpGate.php?ap={主宠id}&bp={副宠id}&p1={道具1}&p2={道具2}&type=do&type1=check`

**约束：**
- 主副宠均 ≥ 40 级
- 操作间隔 ≥ 5 秒
- 消耗 50000 金币
- 宠物未被抽取过成长
- 不可穿装备（需先卸下，否则装备消失）

**合成公式：** 从 `merge` 表查询 `aid={主宠bbId} AND bid={副宠bbId}`

**成功率：**
```
rate = 幸运星/(主czl×2) + (主等级+副等级)/(15×100) + 道具加成 + rand(1,5)/100
```
- 幸运星 ≥ 10 或 主czl ≤ 5 → **100% 成功**
- 成功时随机出 A 阶或 B 阶结果

**成长计算（按主宠成长区间分档）：**

| 主czl 范围 | 公式 |
|-----------|------|
| < 51 | `a_czl + a_lvl/(a_czl+10) + b_lvl·b_czl/200` |
| 51-70 | `a_czl + a_lvl/a_czl + b_lvl·b_czl/350` |
| 70-90 | `a_czl + a_lvl/a_czl + b_lvl·b_czl/500` |
| 90-100 | `a_czl + a_lvl/a_czl + b_lvl·b_czl/700` |
| ≥ 100 | `a_czl + a_lvl/a_czl + b_lvl·b_czl/900` |

所有公式 × `(1 + 道具加成百分比)`，czl 上限 150（神宠 60）

**属性计算：**
```
attr = int((base + mainAttr×mainLvl/400 + subAttr×subLvl/800) × (1 + 道具%))
```

**结果：**
- 成功：两只旧宠删除 → 创建新宠 → 幸运星归零 → 系统公告
- 失败：副宠消失（守护道具可保护） → 幸运星 +1

### 3.3 涅磐 (Tab 3)

**API：** `zsGate.php?ap={主宠}&bp={副宠}&p1={道具1}&p2={道具2}&zs={涅磐兽}`

**约束：**
- 三只宠均 wx ≤ 6、等级 ≥ 60
- 操作间隔 ≥ 10 秒
- 消耗 500000 金币
- 涅磐兽必须是"涅磐兽（亥/午/卯）"且 `muchang=0`
- 宠物未被抽取过成长

**成功率：**
```
rate = round((主等级/30 + 副等级/30) × (1 + npcg道具加成), 2)
rand(1,10000) ≤ rate×100 → 成功
```

**涅磐兽衰减因子：**

| 涅磐兽 | lv 因子 | 效果 |
|--------|--------|------|
| 涅磐兽（卯） | 0.3 | 成长提升最大 |
| 涅磐兽（午） | 0.15 | 中等 |
| 涅磐兽（亥） | 0.05 | 最小 |

**成长公式：**
```
czl = a_czl + round((a_lvl/a_czl/num1 + b_lvl·b_czl/num2) × (1 + lv + 道具npcz加成), 1)
```
num1/num2 根据主宠名称归入 zs1/zs2/zs3 类别不同取值。

**结果：**
- 成功：三只宠全部删除 → 创建新宠 → 系统公告
- 失败：涅磐兽消失（`npbb:1` 道具可保护）

### 3.4 神圣进化 (Tab 4)

**API：** `superJhGate.php?pid={宠物id}&zjsxdj={道具id}`

**约束：**
- 宠物 wx = 7（神圣）
- 进化次数 `remaketimes` < 10
- 从 `super_jh` 表读取每次进化的等级要求和材料
- 金币 = `(zs_progress + remaketimes) × 10000`
- 可选道具 `zjsxdj_*:N%` 按百分比提高属性

**`super_jh` 表结构：**

| 字段 | 说明 |
|------|------|
| pet_id | 宠物基 ID |
| need_levels | 10 个等级的 CSV |
| need_props | 10 组材料的 CSV（格式 `propId:count\|...`） |
| max_czl | 成长上限 |
| zs_progress | 转生阶段 (1-5) |
| max_level | 最大等级 |

**成长计算：**
```
perCzl = rand(ssjh_min×100, ssjh_max×100)
newCzl = current_czl + perCzl/100
```
上限为 `super_jh.max_czl`

**属性重算（7 项）：**
```
attr = round(current × (0.3 + (times+1)/30 + (times+1)×zs_progress/(czl×wxMultiplier)))
```
如果有 `zjsxdj_*` 道具，再 × `(1 + 道具%)`

**结果：** `remaketimes += 1`，等级重置为 1，czl 和属性更新，技能保留。

### 3.5 成长抽取 (Tab 4 内)

**API：** `cqGate.php?pid={宠物id}&pid1={道具1}&pid2={道具2}`

**约束：**
- wx ≤ 6（非神圣），未被抽取过
- 最低成长 30
- 五系宠（wx < 6）必须用道具 3383
- 神宠（wx = 6）不能用道具 3383
- 不能同时用两个 3383

**抽取比例：**

神宠（wx=6）：
| 成长范围 | 比例 |
|---------|------|
| < 65 | 10%-20% |
| 65-85 | 30%-50% |
| 85-100 | 50%-65% |
| 100-110 | 65% |
| 110-115 | 70% |
| 115-120 | 75% |
| ≥ 120 | 80% |

五行宠（wx < 6）：`rand(5%, 15%)`

道具 `inczhl:N` 直接增加比例点数。

**抽取上限：** `min(czl, 600) × 比例%`

金币消耗：`czl × 10000`（czl ≥ 600 时固定 600W）

**结果：**
- 五行宠：宠物删除（uid=0，追加 uid 到名称）
- 神宠：宠物保留，成长变为 1
- 抽取量存入 `player_ext.czl_ss`
- 记录 `player_ext.chouqu_chongwu` 防止重复抽取

---

### 3.6 成长转化 (Tab 4 内)

**API：** `zhGate.php?pid={宠物id}&v={转化值}`

**约束：**
- 仅 wx = 7（神圣）
- v 不能超过 `player_ext.czl_ss` 可用储备
- 宠物需有 `super_jh` 记录

**逻辑：** `czl_ss -= v`，宠物 `czl += v`，上限 `super_jh.max_czl`。超出部分不转化，仅消耗差额。

### 3.7 神圣转生 (Tab 5)

**API：** `sszsInfo.php`

三个操作模式：

| op | URL | 说明 |
|----|-----|------|
| img | `?id={pet_id}&op=img` | 获取可转生的下一个宠物图片列表 |
| str | `?id={super_zs_id}&sid={pet_id}&op=str` | 获取转生需求信息 |
| zs | `?old={petId}&newid={zs_id}&wp1={}&wp2={}&op=zs` | 执行转生 |

**约束（op=zs）：**
- wx = 7（神圣），操作间隔 ≥ 30 秒
- 满足 `super_zs.need_level` 和 `need_czl`
- 拥有 `need_props` 指定材料（格式 `3226|1`）
- 金币 = `zs_progress × 100000`

**`super_zs` 表结构：**

| 字段 | 说明 |
|------|------|
| cur_pet_id | 当前宠物基 ID |
| need_level | 最低转生等级 |
| need_czl | 最低转生成長 |
| need_props | 所需材料 |
| next_pet_id | 结果宠物基 ID |

**成功率：**
```
rate = round(level/30 × (1 + sszs道具加成), 2) × 100
rand(1, 10000) ≤ rate → 成功
```

道具效果：`sszs:X`（增加成功率）、`cszsczlbh:X`（成长倍率）

**属性计算（7 项）：**
```
attr = round((newBase × zs_progress + oldAttr × oldLevel/6000 + oldAttr × oldCzl/9000) × (1 + 道具%))
```

**成长计算：**
```
czl = round(oldCzl × czlMultiplier × 0.01, 1)
```
默认 multiplier = 10

**结果：**
- 成功：旧宠删除 → 创建 next_pet_id 新宠 → mbid 更新
- 失败：仅消耗材料

## 六、后端实现

所有 9 个 API 已实现，位于：

| 文件 | 说明 |
|------|------|
| `PetShrineController.java` | 9 个端点，路径 `/api/pet/**` |
| `PetShrineService.java` | ~800 行，逐一复刻 PHP 7 个文件的公式 |
| `ZsRepository.java` | 新增，涅磐 `zs` 表查询 |

| 端点 | 方法 | PHP 等效 | 说明 |
|------|------|---------|------|
| `/api/pet/{id}/evolve` | POST | `jhGate.php?n=&id=&pids=&bhid=` | 进化，参数 `{style, keepCzlItemId}`，style=1(A进化)/2(B进化) |
| `/api/pet/compose` | POST | `cmpGate.php` | 合成，参数 `{mainPetId, subPetId, item1Id, item2Id}` |
| `/api/pet/nirvana` | POST | `zsGate.php` | 涅磐，参数 `{mainPetId, subPetId, beastId, item1Id, item2Id}` |
| `/api/pet/{id}/sacred-evolve` | POST | `superJhGate.php` | 神圣进化，参数 `{itemId}` |
| `/api/pet/{id}/extract-growth` | POST | `cqGate.php` | 成长抽取，参数 `{item1Id, item2Id}` |
| `/api/pet/{id}/convert-growth` | POST | `zhGate.php` | 成长转化，参数 `{value}` |
| `/api/pet/{id}/sacred-rebirth` | POST | `sszsInfo.php?op=zs` | 神圣转生，参数 `{zsId, item1Id, item2Id}` |
| `/api/pet/rebirth-images` | GET | `sszsInfo.php?op=img` | 转生目标列表，参数 `petId`（后端通过 oldBid 解析模板ID） |
| `/api/pet/{id}/rebirth-info` | GET | `sszsInfo.php?op=str` | 转生需求信息，参数 `zsId` |

### 关键实现细节

- **czl**：直接使用 `userbb.czl` 列的单一浮点值（如 23.2），非 7 项成长属性之和
- **进化 style-1**：索引 remakeid/remakepid/remakelevel 逗号分隔数组（2 个条目对应 2 种风格）
- **合成/Nirvana**：使用 `bb` 表模板名称查找（`$membbname` 缓存），非 `oldBid`
- **Nirvana**：使用 `zs` 表（`aid`/`bid`/`mid`），非 `merge` 表
- **神圣进化**：使用 `wx` 表获取各元素属性乘数系数
- **属性公式**：`intval()` = `(long) Math.floor()`，与 PHP 数学运算完全一致
- **`SkillSysRepository`**：用于解析 `bb.skillist` 的技能定义（`skillsys` 表），新增 `imgeft` 字段映射
- **转生金币**：= 下一个宠物的 `super_jh.zs_progress × 100000`
