## 一、合成机制

### 表结构

`merge(aid, bid, maid, mbid, limits)`

| 字段 | 含义 |
|------|------|
| aid | 主宠模板 ID |
| bid | 副宠模板 ID |
| maid | A 阶结果模板 ID |
| mbid | B 阶结果模板 ID |
| limits | 成长限制（可选），格式 `minMainCzl\|minSubCzl\|maxCzl` |

### 核心规则

- **查询公式**：`SELECT * FROM merge WHERE aid = 主宠模板ID AND bid = 副宠模板ID`
- **A/B 分支**：每次合成默认产 A 阶，B 阶出现概率 = `5% + 道具hechengB加成%`
- **副宠范围**：`bid` 覆盖 1-94 号模板（几乎所有基础/进阶宠都可以当材料）
- **结果可继续合成**：A/B 结果宠的模板 ID 可作为下一轮的主宠 `aid`

### 成功率

```
rate = 幸运星/(主czl×2) + (主等级+副等级)/(15×100) + 道具加成 + rand(1,5)%
幸运星 ≥ 10 或 主czl ≤ 5 → 100%
```

### 成长计算（按主宠成长分档）

| 主czl | 分母 | 上限 |
|-------|------|------|
| < 51 | 200 | 神宠60，普通150 |
| 51-70 | 350 | 神宠60，普通150 |
| 70-90 | 500 | 神宠60，普通150 |
| 90-100 | 700 | 神宠60，普通150 |
| ≥ 100 | 900 | 神宠60，普通150 |
